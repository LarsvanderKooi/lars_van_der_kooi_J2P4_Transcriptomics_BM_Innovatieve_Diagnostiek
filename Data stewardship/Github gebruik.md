**Github gebruik**



In het transcriptomics-project is GitHub gebruikt als omgeving voor het beheren van alle projectbestanden. Het project richt zich op de analyse van RNA-sequentie data met behulp van R.

De repository is opgebouwd met een duidelijke en gestructureerde mappenindeling. De ruwe data wordt apart opgeslagen om de originele informatie te behouden. De bewerkte data worden eveneens apart opgeslagen. Daarnaast zijn er aparte mappen voor scripts, resultaten en documentatie. In de scripts worden alle analyses uitgevoerd, zoals het inladen van data, het uitvoeren van DESeq2-analyses en het maken van visualisaties zoals volcano plots. De files die gemaakt zijn hebben een datum van laatst bewerkt en een versienummer gekregen bijvoorbeeld: GO_results_2026_5_31_v1

GitHub wordt gebruikt voor versiebeheer van het project. Elke wijziging in een R-script of documentatie wordt opgeslagen met een comment. Hierdoor is terug te zien wanneer en waarom aanpassingen zijn gedaan, denk bijvoorbeeld aan het uploaden van een update van het R-script.

Daarnaast draagt GitHub sterk bij aan de reproduceerbaarheid van het onderzoek. Alle scripts en de volledige workflow zijn beschikbaar, waardoor een andere onderzoeker het project kan downloaden en dezelfde analyse opnieuw kan uitvoeren. Dit maakt het onderzoek transparant en controleerbaar.

Ook is er aandacht besteed aan documentatie. In een README-bestand wordt stap voor stap uitgelegd hoe het project is opgebouwd, welke software nodig is en hoe de analyse uitgevoerd moet worden.

Bij het werken met data is zorgvuldig omgegaan met privacy. De data is afkomstig uit bestaand onderzoek en is geanonimiseerd, waardoor het niet mogelijk is om terug te leiden naar individuele personen of patiënten.

Als data steward binnen dit project was ik verantwoordelijk voor het organiseren van bestanden, het documenteren van analyses, het beheren van versies via GitHub en het zorgen voor de reproduceerbaarheid.
